How Som runs Omni day to day

Hey Tim, this is how Som uses Omni

Omni is an execution system, not a demo: Kimi orchestrates, Codex ships heavy work, and Llama runs low-cost heartbeat checks.

3-agent stack

Kimi (brain), Codex (muscle), Llama (heartbeat + hygiene).

2 operating loops

Real-time session loop + background cron and heartbeat loop.

1 rule

Ship complete outputs with validation and clear status reporting.

At a glance

What this setup is optimized for

Speed with guardrails

Fast execution, but with pre-send QA gates and explicit completion receipts.

Parallel heavy lifting

Codex handles coding, file processing, and synthesis while Kimi keeps context and priorities straight.

Quiet autonomy

Heartbeat and cron checks catch urgent items without spamming Som all day.

Tools and automation

Cron, heartbeat, and sub-agent control plane

Cron jobs

Use cron for precise time-based tasks: reminders, exact schedules, and isolated one-shot tasks.

  • Exact schedules: weekly 09:00 reports
  • One-shot reminders: follow up in 20 minutes
  • Isolated jobs with separate model settings

Heartbeat checks

Use heartbeat for grouped periodic checks where slight timing drift is acceptable.

  • Batch email + calendar + mentions in one pass
  • Stay quiet overnight unless urgent
  • Ping only when there is fresh value

Sub-agents

Use this lane when the task needs deep implementation, high context, or long-running synthesis.

  • Always spawn for coding tasks
  • Auto-spawn for multi-file or long-horizon tasks
  • Return completion reports with validation every time

Practical examples

Trigger-to-Outcome Playbooks

Three scenarios below show the full chain: trigger, agent routing, execution actions, and the shipped output.

Scenario 1: Same-day lender document request

Trigger

A lender asks for updated rent roll and insurance proof before a noon cutoff.

Agent routing

Kimi handles priority and approval, then routes data assembly and formatting to Codex.

Actions

  • Collect current lease, ledger, and policy files from property folders
  • Map every value to lender-required fields and highlight missing data
  • Prepare a send-ready summary draft and attach validated artifacts

Output

Decision-ready lender packet plus a concise reply draft for immediate approval and send.

Scenario 2: Rehab scope must hit a hard budget cap

Trigger

A contractor estimate lands above cap and needs same-day value engineering.

Agent routing

Kimi sets tradeoff rules; Codex rewrites line items and verifies totals against cap.

Actions

  • Prioritize by safety, rentability, and return-on-cost impact
  • Rebalance hard cost and contingency to satisfy budget guardrails
  • Cross-check photo evidence against each revised scope line

Output

Final SOW + summary package that is contractor-ready and confirmed within the approved ceiling.

Scenario 3: New automation feature request in a live repo

Trigger

Som requests a feature touching UI, scripts, docs, and deployment flow.

Agent routing

Codex runs implementation and validation loops while Kimi controls scope and outbound safety.

Actions

  • Edit target files with minimal diffs and preserve existing interaction hooks
  • Run lint/build/test and fix regressions before reporting complete
  • Publish with commit hash, validation notes, and live verification

Output

Production-ready release artifacts with reproducible checks and clear handoff details.
